For this to work, we have to know how to call the functions --- ## A Function and Some Tests ```python def sign(value): if value < 0: return -1 else: return 1 def test_sign_negative(): assert sign(-3) == -1 def test_sign_positive(): assert sign(19) == 1 def test_sign_zero(): assert sign(0) == 0 def test_sign_error(): assert sgn(1) == 1 ``` --- ## Running Tests - Apply the function to a __fixture__ - Compare the __actual result__ to the __expected result__ - Each outcome is __pass__, __fail__, or __error__ ```python def run_tests(all_tests): results = {"pass": 0, "fail": 0, "error": 0} for test in all_tests: try: test() results["pass"] += 1 except AssertionError: results["fail"] += 1 except Exception: results["error"] += 1 print(f"pass {results['pass']}") print(f"fail {results['fail']}") print(f"error {results['error']}") ``` --- ## Running Tests ```python TESTS = [ test_sign_negative, test_sign_positive, test_sign_zero, test_sign_error ] run_tests(TESTS) ``` ``` pass 2 fail 1 error 1 ``` - Each test is run exactly the same way - Easy to create new tests - …so long as we remember to add the test function to `TESTS` --- ## How Python Stores Variables - Python stores variables in (something very much like) a dictionary ```python globals() ``` ``` { '__name__': '__main__', '__doc__': None, '__package__': None, '__loader__': , '__spec__': None, '__annotations__': {}, '__builtins__': } ``` --- ## Further Proof ```python my_variable = 123 globals() ``` ``` { '__name__': '__main__', '__doc__': None, '__package__': None, '__loader__': , '__spec__': None, '__annotations__': {}, '__builtins__': , * 'my_variable': 123 } ``` -- - Note: the function `locals` gives local variables --- ## Introspection - We know how to loop over a dictionary's keys ```python def find_tests(prefix): for (name, func) in globals().items(): if name.startswith(prefix): print(name, func) find_tests("test_") ``` ``` test_sign_negative test_sign_positive test_sign_zero test_sign_error ``` -- - Note: when we print a function, Python shows its name and location in memory --- ## Objects Have Attributes - The function `dir` returns a list of an object's attributes ```python def example(): "Docstring for example." print("in example") print(dir(example)) ``` ``` ['__annotations__', '__call__', '__class__', '__closure__', \ '__code__', `__defaults__', '__delattr__', '__dict__', '__dir__', \ '__doc__', '__eq__', '__format__', '__ge__', '__get__', … ] ``` - `__name__`: the function's original name - `__doc__`: its __docstring__ --- ## Classifying Test Functions ```python def test_sign_negative(): assert sign(-3) == -1 *test_sign_negative.skip = True def test_sign_positive(): assert sign(19) == 1 def test_sign_zero(): assert sign(0) == 0 *test_sign_zero.fail = True def test_sign_error(): assert sgn(1) == 1 def classify(func): if hasattr(func, "skip") and func.skip: return "skip" if hasattr(func, "fail") and func.fail: return "fail" return "run" ``` --- ## A Better (?) Test Runner ```python def run_tests(prefix): all_names = [n for n in globals() if n.startswith(prefix)] for name in all_names: func = globals()[name] kind = classify(func) try: if kind == "skip": print(f"skip: {name}") else: func() print(f"pass: {name}") except AssertionError as e: if kind == "fail": print(f"pass (expected failure): {name}") else: print(f"fail: {name} {str(e)}") except Exception as e: print(f"error: {name} {str(e)}") ``` --- ## A Better (?) Test Runner ```python run_tests("test_") ``` ``` skip: test_sign_negative pass: test_sign_positive pass (expected failure): test_sign_zero error: test_sign_error name 'sgn' is not defined ``` - Why the question mark? -- 1. This version prints results instead of returning a data structure - What if we want to compare with previous results? - Or convert to HTML, or…? --- ## Finding Test Files - Test runner should find files containing tests - Typing names each time is as error-prone as typing names of test functions - Use the `glob` module ```python import glob for name in glob.glob("**/*.txt", root_dir=".", recursive=True): print(name) ``` -- - Name comes from an old Unix command that did global matching - Needed so often that it was quickly built into the shell --- class: aside ## Magic - [Clarke's Third Law][clarkes_laws]: any sufficiently advanced technology is indistinguishable from magic - Also true of technologies we encounter before we have the background knowledge they depend on - How we import a module will seem reasonable once we know more - Until then we copy, paste, and move on - Even the most experienced programmers do this in a new domain - It's easier to learn things once we have more context - Setting a problem aside and returning to it later can save a lot of time --- ## Test Case ```python # sample_dir/test_up.py title = "testing example" def test_succeeds(): assert True def test_fails(): assert False ``` -- ```shell python import_example.py sample_dir/test_up.py title ``` ``` After all that, the value is 'testing example' ``` --- ## Loading Files as Modules ```python import importlib.util import sys path = sys.argv[1] variable = sys.argv[2] spec = importlib.util.spec_from_file_location("", path) module = importlib.util.module_from_spec(spec) spec.loader.exec_module(module) value = getattr(module, variable) print(f"After all that, the value is '{value}'") ``` 1.
