--- ## The Parent Class [% inc file="glob_null.py" keep="parent" %] - Assume every child class has a `_match` method - This method returns the location to continue searching - So `Match.match` checks that we've reached the end of the text --- ## The Null Object Class [% inc file="glob_null.py" keep="null" %] - Must be the last one in the chain - Doesn't advance the match (i.e., does nothing) --- ## Refactoring Literal Matcher [% inc file="glob_null.py" keep="lit" %] - Parent class initializes data that it defines - Yes, the `Null` class broke this rule… - Return `None` for "no match" or whatever `self.rest` returns - If this object's `rest` is `Null`, this will be the index after our match --- ## Refactoring Wildcard [% inc file="glob_null.py" keep="any" %] - The exercises will ask, "Why `len(text) + 1`?" --- ## Refactoring Alternatives [% inc file="glob_null.py" keep="either" %] - Could easily be extended to any number of alternatives --- ## Testing - None of the existing tests change - None of the constructors changed - Neither did the signature of `match` - We should (should) add a couple of tests for `Null` - But basically we're done - And we can easily add matchers for other kinds of patterns --- class: summary ## Summary
