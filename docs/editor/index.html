Chapter 7: An Editor

  • FIXME

Terms defined: Delayed Construction pattern

Early text editors were line-at-a-time applications like the debugger of Chapter 20. Most people prefer to use full-screen interactive editors, so this chapters builds a simple one to show how they work. Our design follows Wasim Lorgat‘s tutorial closely, and borrows a few ideas from [Cristian Esquivias‘s ted editor as well.

Our starting point is Python’s curses module. It gives programs access to a library called curses, which provides a uniform interface to terminal applications on several platforms. These applications can do much more than most people realize, and terminal interface libraries like Textualize can do most of what a modern browser can do, but we will stick to the basics for now.

Section 7.1: Logging Keystrokes

Let’s started by writing a terminal application that does nothing except wait for a command to quit:

import curses

def main(scr):
    while True:
        key = scr.getkey()
        if key in "Qq":
            break

if __name__ == "__main__":
    curses.wrapper(main)

Our application is in a function called main. Instead of calling it directly, we pass it to curses.wrapper, which initializes the terminal library for us and then calls our function with an object that represents the terminal. main then goes into a loop that gets a keystroke and quits if the key is either ‘Q’ or ‘q’.

When we run this program our terminal window goes blank and stays that way until we quit.

import curses
import sys

def main(scr, logfile):
    while True:
        key = scr.getkey()
        print(repr(key), file=logfile)
        if key in "Qq":
            break

if __name__ == "__main__":
    assert len(sys.argv) == 2
    with open(sys.argv[1], "w") as writer:
        curses.wrapper(main, writer)

Section 7.3: An Editor Object

  • Functions with lots of parameters are hard to manage
  • But curses.wrapper requires a function
  • Solution: create a class with a __call__ method
    • Must use Delayed Construction design pattern
  • Editor
class Editor:
    def __init__(self):
        self.scr = None
        self.win = None

    def __call__(self, scr, buffer):
        self.setup(scr, buffer)
        self.interact()
  • Setup
    def setup(self, scr, buffer):
        self.scr = scr
        self.win = Window(curses.LINES - 1, curses.COLS - 1)

        self.scr.erase()
        for i, line in enumerate(buffer[:self.win.nrow]):
            self.scr.addstr(i, 0, line[:self.win.ncol])
  • Interaction
    def interact(self):
        while True:
            key = self.scr.getkey()
            if key in "Qq":
                break
  • Launch
if __name__ == "__main__":
    editor = Editor()
    buffer = ['0123456789' * 100] * 1000
    try:
        curses.wrapper(editor, buffer)
    except Exception as exc:
        print(exc)

    def __init__(self):
        self.scr = None
        self.win = None
        self.running = True
        self.actions = {
            "Q": self.quit,

  • But we can only see the top left portion of the file
  • Want to show the available content, which means mapping one coordinate system to another
  • Enhance the Window class to keep track of the top row it’s displaying
    • Keeping track of column will be left as an exercise
class Window:
    def __init__(self, nrow, ncol):
        assert 0 <= nrow and 0 <= ncol
        self.nrow = nrow
        self.ncol = ncol
        self.row = 0

    def bottom(self):
        return self.row + self.nrow - 1

    def up(self, cur):
        if (cur.row == self.row - 1) and (self.row > 0):
            self.row -= 1

    def down(self, buffer, cur):
        if (cur.row == self.bottom() + 1) and (self.bottom() < len(buffer) - 1):
            self.row += 1
  • When we display, we show the rows that are currently visible
    def display(self):
        self.scr.erase()
        visible = self.buffer[self.win.row : (self.win.row + self.win.nrow)]
        for i, line in enumerate(visible):
            self.scr.addstr(i, 0, line[:self.win.ncol])
        self.scr.move(*self.win.translate(self.cur))
  • The translate method turns the cursor’s position into a content position
    def translate(self, cur):
        return cur.row - self.row, cur.col
  • Finally, we adjust the screen when moving
    def up(self):
        if self.cur.row > 0:
            self.cur.row -= 1
        self.limit_col()
        self.win.up(self.cur)

    def down(self):
        if self.cur.row < len(self.buffer) - 1:
            self.cur.row += 1
        self.limit_col()
        self.win.down(self.buffer, self.cur)

class Buffer:
