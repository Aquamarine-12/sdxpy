Chapter 8: Matching Regular Expressions

  • Use regular expressions to match patterns in text and extra data.
  • Have objects control other objects using the Chain of Responsibility design pattern.
  • Use inheritance to make matchers composable.

Terms defined: base class, Chain of Responsibility pattern, eager matching, greedy algorithm, lazy matching, Open-Closed Principle, polymorphism

Sooner or later everyone needs to scrape data out of text files. Regular expressions are the best tool for the job, so this chapter explores how they work by building a simple but extensible pattern matcher. Our approach is inspired by Brian Kernighan's entry in [Oram2007].

Section 8.1: Simple Patterns

Our matcher will initially handle just the five cases shown in Table 8.1. These cases are a small subset of what Python's re module provides, but as Kernighan wrote, "This is quite a useful class; in my own experience of using regular expressions on a day-to-day basis, it easily accounts for 95 percent of all instances."

Table 8.1: Pattern matching cases.
Meaning Character
Any literal character c c
Any single character .
Beginning of input ^
End of input $
Zero or more of something *

Matching is conceptually simple. If the first element of the pattern matches where we are, we see if the rest of the pattern matches what's left; otherwise, we see if the the pattern will match further along. The function users call handles the special case of ^ at the start of a pattern matching the start of the target string being searched.
